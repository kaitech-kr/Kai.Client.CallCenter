# 기본 수칙

## 코드 작업 규칙

1. **절대로 허락없이 임의로 코드를 수정하거나, 삭제하거나, 새로 만들지 않는다**

2. **작업은 한번에 한개의 함수만 한다**

3. **함수가 길때엔 나누어서 확인 받으면서 작업한다**

4. **nul파일을 만들지 않는다**

5. **주석 처리된 함수는 개선점을 찾고 리팩토링할 계획이므로, 항상 개선점을 찾아본 후에 리팩토링한다**

6. **모든 매개변수를 수평(한 줄)으로 작성한다**
   - 예시: `private async Task<StdResult_Status> SearchAndSelectCustomerAsync(IntPtr hWnd고객명, IntPtr hWnd동명, string custName, string chargeName, string fieldName, CancelTokenControl ctrl)`
   - 매개변수가 많아도 수평으로 유지

7. **마우스/키보드 입력은 가능한 한 PostMessage 방식을 사용한다**
   - 이유: 포커스를 잃지 않기 위함 (대규모 리팩토링의 핵심 원칙)

8. **git으로 되돌릴 시 필히 허락을 받고 실행한다**
   - git restore, git reset 등 작업 내용을 되돌리는 명령은 반드시 사전 허가 필요

9. **빌드는 사용자에게 부탁한다**
   - 사용자가 대부분 검증하는데 어차피 빌드하기 때문

10. **간단한 변경(상수 변경, 단순 값 수정 등)은 제안만 하고 직접 수정하지 않는다**
   - 파일 경로와 라인 번호만 알려주고 사용자가 직접 수정하도록 안내
   - 예시: "InsungsAct_RcptRegPage.cs 1099번 줄과 1182번 줄의 `c_nWaitLong`을 `c_nWaitNormal`로 변경하시면 됩니다"
   - 직접 수정하는 경우:
     - 복잡한 로직 변경
     - 여러 파일에 걸친 변경
     - 긴 코드 블록 추가/삭제
